## 🐛 问题描述

用户反馈：上传了头像，但在个人中心页面上没展示出来。

## 🔍 问题分析

### 问题 1: 头像 URL 未通过代理处理

**现象**: 后端返回的头像 URL 是相对路径 `/uploads/image-xxx.png`，前端直接使用无法加载。

**原因**: 移动端运行在 3011 端口，图片 URL 需要通过 Vite 代理访问 3010 端口的静态文件。

```javascript
// ❌ 修复前
<van-image :src="user?.avatarUrl || '...'" />

// ✅ 修复后
<van-image :src="getImageUrl(user?.avatarUrl) || '...'" />
```

### 问题 2: 保存后未更新 userStore

**现象**: 保存成功后，页面上的头像和昵称没有立即更新。

**原因**: `saveProfile` 函数只调用了 `loadUserData()`，但没有直接更新 `userStore.user`，导致响应式更新延迟。

```javascript
// ❌ 修复前
await updateProfile(data)
await loadUserData()  // 异步，有延迟

// ✅ 修复后
const res = await updateProfile(data)
userStore.user = { ...res.data.user }  // 立即更新
user.value = { ...res.data.user }  // 立即更新
await loadUserData()  // 确保数据一致
```

## ✅ 修复方案

### 1️⃣ 添加图片 URL 处理函数

**新增函数**:
```javascript
const getImageUrl = (url) => {
  if (!url) return ''
  // 如果已经是完整 URL，直接返回
  if (url.startsWith('http://') || url.startsWith('https://') || url.startsWith('data:')) {
    return url
  }
  // 相对路径，通过 Vite 代理访问
  return url
}
```

**使用**:
```vue
<!-- 用户卡片头像 -->
<van-image
  :src="getImageUrl(user?.avatarUrl) || 'https://via.placeholder.com/80'"
  width="60"
  height="60"
  round
/>

<!-- 编辑对话框头像预览 -->
<div class="avatar-preview" v-if="editForm.avatarUrl">
  <van-image
    :src="getImageUrl(editForm.avatarUrl)"
    width="80"
    height="80"
    round
    fit="cover"
  />
</div>
```

### 2️⃣ 立即更新 userStore

**修改 saveProfile 函数**:
```javascript
const saveProfile = async () => {
  try {
    // 验证昵称
    if (!editForm.value.nickname || editForm.value.nickname.trim() === '') {
      showToast('昵称不能为空')
      return
    }
    
    const res = await updateProfile({
      nickname: editForm.value.nickname.trim(),
      phone: editForm.value.phone,
      avatarUrl: editForm.value.avatarUrl
    })
    
    // ✅ 立即更新 userStore
    if (res.data.user) {
      userStore.user = { ...res.data.user }
      user.value = { ...res.data.user }
    }
    
    showToast('保存成功')
    showEditDialog.value = false
    await loadUserData()  // 确保数据一致
  } catch (error) {
    console.error('保存失败:', error)
    showToast(error.message || '保存失败')
  }
}
```

### 3️⃣ 添加错误提示

**改进错误处理**:
```javascript
catch (error) {
  console.error('保存失败:', error)
  showToast(error.message || '保存失败')  // 显示具体错误信息
}
```

## ⚠️ 注意事项

### 1. Vite 代理配置

确保 `vite.config.js` 中有 `/uploads` 代理：

```javascript
proxy: {
  '/uploads': {
    target: 'http://localhost:3010',
    changeOrigin: true
  }
}
```

### 2. 后端静态文件服务

确保后端配置了静态文件服务：

```javascript
app.use('/uploads', express.static(path.join(__dirname, '../uploads')));
```

### 3. 响应式更新

使用展开运算符创建新对象，确保 Vue 的响应式系统能检测到变化：

```javascript
userStore.user = { ...res.data.user }  // ✅ 创建新对象
userStore.user = res.data.user          // ❌ 可能不触发更新
```
