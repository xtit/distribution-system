### 2️⃣ 后端 API

#### 新增接口

**上传头像**
- **路由**: `POST /api/auth/upload-avatar`
- **认证**: 需要 JWT Token
- **中间件**: multer 文件上传
- **参数**: `avatar` (File)
- **返回**: `{ url: '/uploads/image-xxx.png' }`

**更新个人资料**
- **路由**: `PUT /api/auth/profile`
- **认证**: 需要 JWT Token
- **参数**: 
  - `nickname` (String)
  - `phone` (String)
  - `email` (String)
  - `avatarUrl` (String)
- **返回**: 更新后的用户信息

## 🔧 技术实现

### 前端技术

**组件**:
- `van-uploader` - 图片上传
- `van-field` - 表单输入
- `van-dialog` - 弹窗
- `van-image` - 图片显示

**状态管理**:
```javascript
const editForm = ref({
  nickname: '',
  phone: '',
  avatarUrl: ''
})
const avatarFileList = ref([])
```

**上传逻辑**:
```javascript
const onAvatarRead = async (file) => {
  showLoadingToast({ message: '上传中...' })
  const res = await uploadAvatar(file.file)
  editForm.value.avatarUrl = res.data.url
  showToast('上传成功')
}
```

### 后端技术

**中间件**:
- `multer` - 文件上传处理
- `authMiddleware` - JWT 认证

**上传配置**:
```javascript
const storage = multer.diskStorage({
```

## 📊 数据流程

### 头像上传流程

```
用户选择图片
  ↓
前端验证（格式、大小）
  ↓
FormData 封装
  ↓
POST /api/auth/upload-avatar
  ↓
后端验证
  ↓
保存到 uploads/
  ↓
返回 URL: /uploads/image-xxx.png
  ↓
前端显示预览
```

### 保存资料流程

```
用户点击确认
  ↓
前端验证（昵称非空）
  ↓
PUT /api/auth/profile
  ↓
{ nickname, phone, avatarUrl }
  ↓
后端更新数据库
  ↓
返回更新后的用户信息
  ↓
前端刷新显示
```
