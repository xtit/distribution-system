# 变量命名冲突修复报告

**修复时间**: 2026-03-21 11:10  
**修复人**: 徐小呆

---

## 🐛 问题描述

移动端订单页面报错：
```
[plugin:vite:vue] [vue/compiler-sfc] Identifier 'confirmReceive' has already been declared.
```

## 🔍 问题分析

### 错误原因

**变量命名冲突**：

1. 从 API 导入了 `confirmReceive` 函数：
```javascript
import { confirmReceive } from '@/api/product'
```

2. 组件内定义了同名的 `confirmReceive` 函数：
```javascript
const confirmReceive = async (orderId) => { ... }
```

3. Vue 编译器报错：**同一个作用域内不能有两个同名的标识符**

### 错误位置

**文件**: `mobile-frontend/src/views/OrderList.vue`  
**行号**: 89 行和 210 行

---

## ✅ 修复方案

### 方案：重命名导入的 API 函数

使用 `as` 关键字给导入的函数起别名：

```javascript
// 修复前
import { confirmReceive } from '@/api/product'

// 修复后
import { confirmReceive as confirmReceiveAPI } from '@/api/product'
```

### 修改调用

```javascript
// 修复前（调用自己，导致递归）
const confirmReceive = async (orderId) => {
  await confirmReceive(orderId)  // ❌ 调用自己
}

// 修复后
const confirmReceive = async (orderId) => {
  await confirmReceiveAPI(orderId)  // ✅ 调用 API 函数
}
```

## 🎯 命名规范建议

### 避免命名冲突的最佳实践

1. **API 函数加后缀**：
   - `confirmReceiveAPI`
   - `getOrdersAPI`
   - `createOrderAPI`

2. **或者组件函数加前缀**：
   - `handleConfirmReceive`
   - `handleGetOrders`
   - `handleCreateOrder`

3. **推荐方案**：
```javascript
// 导入时加 API 后缀
import { confirmReceive as confirmReceiveAPI } from '@/api/product'

// 组件内使用 handle 前缀
const handleConfirmReceive = async (orderId) => {
  await confirmReceiveAPI(orderId)
}
```

## 📝 经验总结

### 问题根源

在修复"确认收货功能失效"时，添加了 API 导入但忽略了命名冲突：

```javascript
// ❌ 错误写法
import { confirmReceive } from '@/api/product'  // 导入

const confirmReceive = async (orderId) => { ... }  // 定义（冲突！）
```

### 避免方法

1. **导入时检查**: 导入新函数时检查是否已有同名变量
2. **使用别名**: 使用 `as` 关键字避免冲突
3. **统一规范**: 建立命名规范（如 API 函数加 `API` 后缀）
4. **代码审查**: 提交前检查是否有命名冲突
