## 📋 下一步操作

### 方案 A：快速部署（推荐新手）

**使用 Railway/Render 部署后端**（更简单，无需修改代码）

1. 访问 https://railway.app
2. 创建新项目
3. 选择 "Deploy from GitHub repo"
4. 选择 `xtit/distribution-system` 仓库
5. 设置 Root Directory: `backend`
6. 添加环境变量：
   - `DATABASE_URL`: Railway 自动提供 PostgreSQL
   - `JWT_SECRET`: 随机字符串
7. 部署完成后，复制 Railway 提供的 URL
8. 更新前端 API 地址：
   - 移动端：修改 `mobile-frontend/.env.production`
   - 管理后台：修改 `admin-frontend/index.html`

### 方案 B：Vercel + Supabase（推荐）

**优势**: 全栈 Vercel，性能更好

#### 步骤 1: 创建 Supabase 数据库
1. 访问 https://supabase.com
2. 创建新项目（免费）
3. 复制 Connection String (Pooler 模式)

#### 步骤 2: 初始化数据库
1. 进入 Supabase SQL Editor
2. 运行 `backend/supabase-init.sql`

#### 步骤 3: 部署后端到 Vercel
```bash
cd backend
vercel --prod
```

#### 步骤 4: 配置环境变量
在 Vercel 项目设置中添加：
- `DATABASE_URL`
- `JWT_SECRET`

#### 步骤 5: 测试 API
```bash
curl https://distribution-system-api.vercel.app/health
```

## ⚠️ 注意事项

### Vercel Serverless 限制
1. **冷启动**: 首次访问 1-3 秒延迟
2. **执行时间**: 最大 10 秒（Hobby 套餐）
3. **文件大小**: 上传限制 4.5MB
4. **无状态**: 不能保存文件到本地

### 数据库优化
1. 使用 Supabase 连接池
2. 为常用查询字段添加索引
3. 生产环境强制 SSL 连接

### 安全建议
1. 修改默认管理员密码
2. 设置强 JWT_SECRET
3. 启用 CORS 白名单
4. 定期备份数据库
