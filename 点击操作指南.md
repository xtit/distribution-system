# 🚀 Railway 部署 - 点击操作指南

## 请按顺序点击以下链接（每个链接打开一个新标签页）

### 第 1 步：打开 Railway 创建项目页面
🔗 **点击这里**：https://railway.app/new?workspaceId=5695b3b9-7af4-4620-8897-1e2c6112fe08

---

### 第 2 步：选择 GitHub 部署
在新打开的页面中：
1. 点击 **"GitHub Repository"** 选项
2. 如果提示授权，点击 **"Authorize Railway"**

---

### 第 3 步：选择仓库
在 GitHub 仓库列表中：
1. 找到 **`xtit/distribution-system`**
2. 点击它

---

### 第 4 步：等待部署开始
Railway 会自动开始部署，然后：
1. 点击服务卡片进入详情页

---

### 第 5 步：配置 Root Directory
在服务详情页：
1. 点击顶部 **"Settings"** 标签
2. 找到 **"Root Directory"** 输入框
3. 输入：`backend`
4. 点击 **"Save"** 按钮

---

### 第 6 步：添加环境变量
1. 点击顶部 **"Variables"** 标签
2. 点击 **"New Variable"** 按钮
3. 依次添加以下 4 个变量：

**变量 1 - DATABASE_URL:**
```
Name: DATABASE_URL
Value: postgresql://postgres.kdpgnhpfkyugrlnpmtju:JgdDwJ9iK2Df8hdu@aws-1-ap-northeast-1.pooler.supabase.com:6543/postgres
```
点击 **"Add Variable"** 保存

**变量 2 - JWT_SECRET:**
```
Name: JWT_SECRET
Value: 51fd80b0b242824b750c231983b512ff1dc38f046c1da06206f4a54b70d35233
```
点击 **"Add Variable"** 保存

**变量 3 - NODE_ENV:**
```
Name: NODE_ENV
Value: production
```
点击 **"Add Variable"** 保存

**变量 4 - PORT:**
```
Name: PORT
Value: 3010
```
点击 **"Add Variable"** 保存

---

### 第 7 步：等待部署完成
1. 点击顶部 **"Deployments"** 标签
2. 等待部署状态变为 **"Running"**（绿色）
3. 大约需要 2-3 分钟

---

### 第 8 步：获取 API 地址
部署成功后：
1. 点击顶部 **"Settings"** 标签
2. 滚动到 **"Domains"** 或 **"Public Networking"** 部分
3. 复制显示的 URL（格式：`https://xxx-production.up.railway.app`）

---

### 第 9 步：测试 API
在浏览器中打开：
```
https://your-railway-url.up.railway.app/health
```

应该看到：
```json
{
  "status": "ok",
  "timestamp": "2026-03-23T..."
}
```

## ❓ 遇到问题？

### 找不到 Root Directory 设置？
- 确保你点击了服务卡片进入详情页
- 然后点击 "Settings" 标签

### 部署失败？
- 检查环境变量是否正确
- 查看 "Deployments" 标签页的日志
