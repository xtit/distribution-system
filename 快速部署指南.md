---

### 2️⃣ 运行一键部署脚本（5 分钟）

**在本地执行**:

```bash
cd /home/admin/openclaw/workspace/distribution-system

# 1. 修改部署脚本
nano deploy-aliyun.sh

# 修改这两行:
SERVER_IP="你的服务器 IP"  # 改成你的阿里云 IP
SERVER_USER="root"

# 2. 添加执行权限
chmod +x deploy-aliyun.sh

# 3. 运行部署
./deploy-aliyun.sh
```

**脚本会自动**:
- ✅ 连接服务器
- ✅ 安装 Node.js 和 PM2
- ✅ 上传后端代码
- ✅ 安装依赖
- ✅ 启动服务
- ✅ 构建前端

---

### 3️⃣ 部署前端到 Cloudflare（3 分钟）

**步骤**:

1. **访问**: https://pages.cloudflare.com

2. **登录/注册** Cloudflare 账号

3. **创建项目**:
   - 点击 "Create a project"
   - 选择 "Direct Upload"

4. **上传文件**:
   - 打开文件夹：`mobile-frontend/dist/`
   - 全选所有文件（Ctrl+A）
   - 拖拽到 Cloudflare 上传区域

5. **设置项目名称**: `distribution-frontend`

6. **点击 "Deploy"**

**获得地址**: `https://distribution-frontend.pages.dev`

---

### 4️⃣ 配置 API 地址（2 分钟）

**方式一：修改前端代码（简单）**

```bash
# 在本地修改
cd mobile-frontend/src/api

# 编辑 request.js
nano request.js

# 修改 baseURL:
const request = axios.create({
  baseURL: 'http://你的服务器 IP:3010/api',
  timeout: 10000
})

# 重新构建
cd ..
npm run build

# 重新上传 dist 到 Cloudflare
```

**方式二：Cloudflare 环境变量（推荐）**

1. 进入 Cloudflare Pages 项目
2. Settings → Environment variables
3. 添加：
   ```
   VITE_API_BASE = http://你的服务器 IP:3010/api
   ```
4. 点击 "Redeploy"

## 🔧 常见问题

### Q1: 页面打不开？

**检查**:
```bash
# 1. 服务器是否运行
ssh root@你的服务器 IP
pm2 status

# 2. 端口是否开放
curl http://localhost:3010/health
```

### Q2: 图片上传失败？

**检查**:
```bash
# 查看上传目录权限
ls -lh /var/www/distribution-system/backend/uploads/

# 修改权限
chmod 755 uploads/
```

### Q3: Cloudflare 访问慢？

**解决**:
- Cloudflare 国内节点较少，属正常现象
- 可考虑购买域名并配置 CDN

## 🎯 下一步优化

### 1. 配置域名（可选）

**购买域名**:
- 阿里云域名：https://wanwang.aliyun.com/domain/
- 推荐：`.com` ¥55/年 或 `.top` ¥10/年

**配置步骤**:
1. 在 Cloudflare Pages 添加自定义域名
2. 按照提示配置 DNS
3. 自动获得 HTTPS

### 2. 配置 HTTPS（推荐）

```bash
# 在服务器上安装 Certbot
apt install -y certbot python3-certbot-nginx

# 获取证书（需要域名）
certbot --nginx -d your-domain.com
```

### 3. 设置自动备份

```bash
# 创建备份脚本
cat > /var/www/distribution-system/backup.sh << 'EOF'
#!/bin/bash
cd /var/www/distribution-system/backend
tar -czf backup-$(date +%Y%m%d).tar.gz data/
# 保留最近 7 天备份
find . -name "backup-*.tar.gz" -mtime +7 -delete
EOF

chmod +x backup.sh

# 添加定时任务
crontab -e
# 添加：0 2 * * * /var/www/distribution-system/backup.sh
```
