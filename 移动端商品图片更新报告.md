### 2️⃣ 问题分析

**问题根源**: 数据库中商品的 `main_image` 字段为 `null`

```sql
-- 修复前
SELECT id, name, main_image FROM products;
-- 结果：大部分商品 main_image 为 NULL
```

**原因**: 之前商品创建时没有上传图片，导致主图为空。

### 3️⃣ 数据修复

**创建更新脚本**: `backend/src/scripts/update-product-images.js`

```javascript
// 更新所有主图为 null 的商品
const [updateCount] = await sequelize.query(`
  UPDATE products 
  SET main_image = :imageUrl 
  WHERE main_image IS NULL
`, {
  replacements: { imageUrl },
  type: sequelize.QueryTypes.UPDATE
});
```

**执行结果**:
```

📝 图片 URL: /uploads/image-1774060611340-212405546.png
```

## ⚠️ 注意事项

### 1. 图片路径
- 后端返回相对路径：`/uploads/image-xxx.png`
- 移动端需要能访问后端静态文件
- 确保 CORS 配置正确

### 2. 图片大小
- 列表页：100x100 缩略图
- 详情页：100% 宽度大图
- 建议使用 CDN 或图片服务优化

### 3. 占位图
- 当 `mainImage` 为 null 时使用占位图
- 当前使用：`https://via.placeholder.com/200`
- 建议：使用本地占位图或默认商品图
