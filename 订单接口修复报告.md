# 订单页接口 404 问题修复报告

**修复时间**: 2026-03-19 19:25 (GMT+8)

## 🐛 问题描述

用户报告订单页接口返回 404 错误。

## 🔍 问题原因

**路由顺序错误** - Express 路由按定义顺序匹配，`/:id` 动态参数路由在 `/orders` 之前定义，导致 `/api/products/orders` 被匹配为 `/api/products/:id`，其中 `id = 'orders'`。

### 错误的路由配置
```javascript
// ❌ 错误顺序
router.get('/:id', productController.getProductDetail);  // 先定义
router.get('/orders', authMiddleware, productController.getOrders);  // 后定义
```

当请求 `/api/products/orders` 时：
1. Express 先匹配 `/:id` 路由
2. `'orders'` 被当作商品 ID
3. 查询商品表中 `id = 'orders'` 的记录
4. 返回 404 "商品不存在"

## ✅ 修复方案

调整路由顺序，将具体路由放在动态参数路由之前：

```javascript
// ✅ 正确顺序
// 1. 公开路由（商品列表）
router.get('/', productController.getProducts);

// 2. 订单相关路由（必须在 /:id 之前定义）
router.post('/orders', authMiddleware, productController.createOrder);
router.get('/orders', authMiddleware, productController.getOrders);
router.post('/orders/:id/payment', authMiddleware, productController.confirmPayment);
router.post('/orders/:id/complete', authMiddleware, productController.completeOrder);

// 3. 商品详情路由（放在最后，避免匹配其他路径）
router.get('/:id', productController.getProductDetail);
```

## 🔧 附加修复

在修复过程中发现并修复了以下相关字段映射问题：

### 1. Order 模型排序字段
**问题**: `orderBy` 配置缺失
**修复**: 添加 `orderBy: 'created_at'` 配置

### 2. 控制器中的字段名
**问题**: 使用驼峰命名 `createdAt` 而非数据库字段名 `created_at`
**修复位置**:
- `productController.js` - getOrders 函数
- `authController.js` - getReferees 函数

```javascript
// ❌ 错误
order: [['createdAt', 'DESC']]
attributes: ['createdAt']

// ✅ 正确
order: [['created_at', 'DESC']]
attributes: ['created_at']
```

## 🎯 Express 路由匹配规则

**重要原则**: 路由定义顺序非常重要！

1. **具体路由优先** - `/orders` 必须在 `/:id` 之前
2. **动态参数放最后** - `/:id` 会匹配任何值
3. **带参数的路由** - `/orders/:id` 必须在 `/orders` 之后

```javascript
// ✅ 正确示例
router.get('/orders', handler);      // 具体路径
router.get('/orders/:id', handler);  // 带参数的具体路径
router.get('/:id', handler);         // 通用动态参数

// ❌ 错误示例
router.get('/:id', handler);         // 会匹配所有路径
router.get('/orders', handler);      // 永远不会被执行
```
