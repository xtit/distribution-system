# 📋 手动执行步骤指南

## ⚠️ 重要说明

由于 Supabase SQL Editor 需要手动粘贴执行 SQL 脚本，请按照以下步骤操作：

## 步骤 1: 初始化数据库（必须）

### 1.1 打开 SQL Editor
浏览器标签页中已打开：
- URL: https://supabase.com/dashboard/project/kdpgnhpfkyugrlnpmtju/sql

### 1.2 创建新查询
1. 点击左侧 "New query" 按钮（或页面中的 "New" 标签）

### 1.3 复制 SQL 脚本
1. 在本地打开文件：`/home/admin/openclaw/workspace/distribution-system/backend/supabase-init.sql`
2. 全选内容：`Ctrl + A`
3. 复制：`Ctrl + C`

### 1.4 粘贴并执行
1. 在 SQL Editor 文本框中粘贴：`Ctrl + V`
2. 点击 "Run" 按钮（或按 `Ctrl + Enter`）

### 1.5 确认成功
应该看到类似输出：
```
CREATE TABLE

CREATE TABLE
CREATE TABLE
CREATE TABLE
CREATE TABLE
CREATE INDEX
...
INSERT 0 1
```

## 步骤 2: 配置 Vercel 环境变量（必须）

### 方法 A: 使用 CLI（推荐）

在终端执行以下命令：

```bash
cd /home/admin/openclaw/workspace/distribution-system/backend

# 1. 添加数据库连接
vercel env add DATABASE_URL production
# 当提示时粘贴：
postgresql://postgres.kdpgnhpfkyugrlnpmtju:JgdDwJ9iK2Df8hdu@aws-1-ap-northeast-1.pooler.supabase.com:6543/postgres

# 2. 添加 JWT 密钥
vercel env add JWT_SECRET production
# 当提示时粘贴：
51fd80b0b242824b750c231983b512ff1dc38f046c1da06206f4a54b70d35233

# 3. 添加环境
vercel env add NODE_ENV production
# 当提示时输入：
production

# 4. 重新部署
vercel --prod
```

2. 点击 "Add New..."
