## 🐛 问题描述

用户反馈：邀请海报页面中，二维码图片显示为空白占位符。

**现象**:
- ✅ 用户信息正常显示
- ✅ 佣金余额正常显示
- ❌ 二维码图片显示为空白

---

## 🔍 问题诊断

### 1. 检查 API 返回

```bash
curl /api/auth/me -H "Authorization: Bearer TOKEN"

# 返回：
{
  "qrCodeUrl": "/qrcodes/user_1_1774336614037.png"  // 相对路径
}
```

### 2. 检查前端代码

`InvitePoster.vue`:
```vue
<van-image :src="qrCodeUrl" width="200" height="200" />
```

```javascript
// 问题代码
qrCodeUrl.value = user.value?.qrCodeUrl || 'https://via.placeholder.com/200'
```

**问题**: 
- 后端返回的是相对路径：`/qrcodes/user_1_xxx.png`
- 前端直接使用，导致请求到 Vercel（前端域名）而不是 Railway（后端域名）
- 正确应该是：`https://distribution-system-production.up.railway.app/qrcodes/user_1_xxx.png`

---

## ✅ 修复方案

### 修改文件
`mobile-frontend/src/views/InvitePoster.vue`

### 修复代码

```javascript
// 后端 API 基础 URL
const API_BASE_URL = import.meta.env.VITE_API_URL || 'https://distribution-system-production.up.railway.app'

const loadUserData = async () => {
  try {
    await userStore.fetchCurrentUser()
    user.value = userStore.user
    
    // 处理二维码 URL：如果是相对路径，拼接完整 URL
    if (user.value?.qrCodeUrl) {
      if (user.value.qrCodeUrl.startsWith('/')) {
        qrCodeUrl.value = API_BASE_URL + user.value.qrCodeUrl
      } else {
        qrCodeUrl.value = user.value.qrCodeUrl
      }
    } else {
      qrCodeUrl.value = 'https://via.placeholder.com/200'
    }
  } catch (error) {
    console.error('加载用户信息失败:', error)
    showToast('加载失败')
  }
}
```

### 修复逻辑

1. 检查 `qrCodeUrl` 是否以 `/` 开头（相对路径）
2. 如果是相对路径，拼接后端 API 基础 URL
3. 如果是完整 URL（http/https 开头），直接使用
4. 如果为空，使用占位图
