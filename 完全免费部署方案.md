## 📋 部署策略


**优点**:
- 后端不休眠
- 配置简单

**缺点**:
- 需要两个平台

## ⚡ 最快方案：Vercel + Hugging Face

### 第一步：准备代码（2 分钟）

#### 1. 修改后端配置

创建 Vercel 配置文件：

```javascript
// vercel.json
{
  "version": 2,
  "builds": [
    {
      "src": "backend/src/index.js",
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "backend/src/index.js"
    }
  ]
}
```

#### 2. 修改数据库配置

```javascript
// backend/src/config/database.js
const path = require('path');

// Vercel 环境使用内存数据库或外部数据库
const isVercel = process.env.VERCEL;

if (isVercel) {
  // Serverless 环境使用 SQLite 内存模式或外部服务
  // 推荐使用免费的 Supabase PostgreSQL
  const sequelize = new Sequelize(
    process.env.DATABASE_URL || 'sqlite::memory:',
    {
      dialect: 'postgres',
      dialectOptions: {
        ssl: { require: true, rejectUnauthorized: false }
      }
    }
  );
} else {
  // 本地开发使用 SQLite 文件
  const sequelize = new Sequelize({
    dialect: 'sqlite',
    storage: path.join(__dirname, '../../data/dev.db')
  });
}
```

#### 3. 修改前端 API 地址

```javascript
// mobile-frontend/src/api/request.js
const API_BASE = process.env.VITE_API_BASE || 
                 'https://your-app.vercel.app/api';

const request = axios.create({
  baseURL: API_BASE,
  timeout: 10000
});
```

### 第二步：部署到 Vercel（3 分钟）

#### 方法 1：通过 GitHub（推荐）

1. **创建 GitHub 仓库**
   ```bash
   cd /home/admin/openclaw/workspace/distribution-system
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的用户名/distribution-system.git
   git push -u origin main
   ```

2. **部署到 Vercel**
   - 访问：https://vercel.com/new
   - 点击 "Continue with GitHub"
   - 导入你的仓库
   - 配置：
     ```
     Framework Preset: Vite
     Root Directory: mobile-frontend
     Build Command: npm run build
     Output Directory: dist
     ```
   - 添加环境变量：
     ```
     VITE_API_BASE = https://your-app.vercel.app/api
     ```
   - 点击 "Deploy"

#### 方法 2：直接上传（无需 Git）

1. **安装 Vercel CLI**
   ```bash
   npm install -g vercel
   ```

2. **部署前端**
   ```bash
   cd mobile-frontend
   npm run build
   vercel --prod
   ```

3. **部署后端**
   ```bash
   cd backend
   vercel --prod
   ```

## 🎯 我的推荐：Vercel + SQLite（简化版）

### 配置步骤

#### 1. 创建 vercel.json

```json
{
  "version": 2,
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "/src/index.js"
    }
  ],
  "env": {
    "NODE_ENV": "production",
    "DATABASE_PATH": "/tmp/database.db"
  }
}
```

#### 2. 修改后端代码

```javascript
// backend/src/index.js
const express = require('express');
const path = require('path');
const cors = require('cors');

const app = express();

app.use(cors());
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// 静态文件（上传的图片等）
app.use('/uploads', express.static(path.join(__dirname, '../uploads')));

// API 路由
app.use('/api/auth', require('./routes/auth'));
app.use('/api/products', require('./routes/products'));
app.use('/api/admin', require('./routes/admin'));

// 健康检查
app.get('/health', (req, res) => {
  res.json({ status: 'ok', timestamp: new Date().toISOString() });
});

// Vercel 导出
module.exports = app;

// 本地开发时启动服务器
if (process.env.VERCEL !== '1') {
  const PORT = process.env.PORT || 3010;
  app.listen(PORT, () => {
    console.log(`✅ 服务器启动成功：http://localhost:${PORT}`);
  });
}
```

#### 3. 部署

```bash
# 安装 Vercel CLI
npm install -g vercel

# 登录
vercel login

# 部署后端
cd backend
vercel --prod

# 部署前端
cd ../mobile-frontend
vercel --prod
```

## ⚠️ 重要提示

### Serverless 的限制

1. **冷启动**: 第一次访问需要 2-3 秒
2. **执行时间**: 每个请求最多 10 秒
3. **内存**: 最多 1024MB
4. **数据库**: SQLite 在 Serverless 下有限制

### 推荐方案

如果你的应用：
- **用户少**（<100 人/天）→ Vercel + SQLite
- **需要数据库** → Vercel + Supabase
- **只需展示** → Static.app
