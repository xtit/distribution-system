# 📱 移动端二维码生成功能修复报告

**修复时间**: 2026-03-24 15:17 GMT+8  
**问题**: 移动端二维码生成功能失效  
**状态**: ✅ 已修复

---

## 🐛 问题描述

用户反馈：移动端点击"生成二维码"按钮时，应该跳转到新的邀请海报页面，但功能失效了。

**预期效果**:
- 点击"分享二维码"按钮
- 跳转到 `/invite-poster` 页面
- 显示用户信息、佣金余额、二维码海报

---

## 🔍 问题诊断

### 1. 检查前端路由

✅ 路由配置正确：
```javascript
{
  path: '/invite-poster',
  name: 'InvitePoster',
  component: () => import('@/views/InvitePoster.vue')
}
```

✅ 页面组件存在：`InvitePoster.vue`

### 2. 检查前端逻辑

`Profile.vue` 中的 `showQRCode` 函数：
```javascript
const showQRCode = () => {
  if (user.value?.qrCodeUrl) {
    router.push('/invite-poster')  // 有二维码才跳转
  } else {
    showToast('二维码生成中')  // 没有二维码只显示提示
  }
}
```

**问题**: 只有当 `user.qrCodeUrl` 存在时才跳转，否则只显示"二维码生成中"。

### 3. 检查后端 API

```bash
curl /api/auth/me -H "Authorization: Bearer TOKEN"

# 返回：
{
  "qrCodeUrl": null,  // ❌ 问题所在
  "referralCode": "ADMIN001"
}
```

**根本原因**: 
- 线上数据库的用户 `qrCodeUrl` 字段为 `null`
- 数据同步时只同步了数据库记录，没有同步二维码文件
- 导致前端判断 `qrCodeUrl` 为空，不跳转页面

---

## ✅ 修复方案

### 后端修复

修改 `backend/src/controllers/authController.js` 的 `getCurrentUser` 函数：

```javascript
exports.getCurrentUser = async (req, res) => {
  try {
    const user = await User.findByPk(req.user.id, {
      attributes: { exclude: ['passwordHash'] }
    });

    // 如果用户没有二维码，自动生成
    if (!user.qrCodeUrl) {
      console.log(`用户 ${user.id} 没有二维码，正在生成...`);
      const qrCodePath = await generateQRCode(user);
      user.qrCodeUrl = qrCodePath;
      await user.save();
      console.log(`用户 ${user.id} 二维码生成成功：${qrCodePath}`);
    }

    res.json({
      code: 200,
      data: { user }
    });
  } catch (error) {
    console.error('获取用户信息失败:', error);
    res.status(500).json({
      code: 500,
      message: '获取用户信息失败',
      error: error.message
    });
  }
};
```

### 修复逻辑

1. 用户请求获取个人信息时
2. 检查 `qrCodeUrl` 是否为空
3. 如果为空，自动调用 `generateQRCode` 生成二维码
4. 保存到数据库并返回给前端
5. 前端收到 `qrCodeUrl` 后，点击按钮正常跳转
