## 🌟 方案一：Vercel + Render（推荐）

### 优势
- ✅ **完全免费**
- ✅ **自动 HTTPS**
- ✅ **自动部署**（Git 推送即部署）
- ✅ **国内访问速度尚可**
- ✅ **配置简单**

### 前端部署到 Vercel

**网址**: https://vercel.com

**步骤**:
1. 注册账号（可用 GitHub 登录）
2. 点击 "Add New Project"
3. 导入你的 GitHub 仓库
4. 配置构建设置：
   ```
   Framework Preset: Vite
   Root Directory: mobile-frontend
   Build Command: npm run build
   Output Directory: dist
   ```
5. 点击 "Deploy"

**环境变量**:
```
VITE_API_BASE=https://your-app.onrender.com/api
```

**特点**:
- 自动分配域名：`your-app.vercel.app`
- 支持自定义域名
- 全球 CDN 加速
- 每月 100GB 流量免费

### 后端部署到 Render

**网址**: https://render.com

**步骤**:
1. 注册账号（可用 GitHub 登录）
2. 点击 "New +" → "Web Service"
3. 导入你的 GitHub 仓库
4. 配置：
   ```
   Name: distribution-backend
   Region: Singapore (推荐，离中国近)
   Root Directory: backend
   Runtime: Node
   Build Command: npm install
   Start Command: node src/index.js
   Instance Type: Free
   ```

**环境变量**:
```
PORT=3010
NODE_ENV=production
DATABASE_URL=./data/production.db
JWT_SECRET=your-secret-key-change-this
```

**特点**:
- 自动分配域名：`your-app.onrender.com`
- 免费套餐：750 小时/月（足够单用户使用）
- 自动 HTTPS
- 休眠策略：15 分钟无访问自动休眠（下次访问唤醒约 30 秒）

### 数据库方案

**方案 A: SQLite（简单）**
```javascript
// backend/src/config/database.js
const database = new Sequelize({
  dialect: 'sqlite',
  storage: process.env.DATABASE_PATH || './data/production.db'
});
```

**方案 B: PostgreSQL（推荐生产环境）**
- Render 免费 PostgreSQL 数据库
- 每月 1GB 存储
- 90 天自动备份

**网址**: https://netlify.com

1. 注册账号
2. 拖拽 `mobile-frontend/dist` 文件夹到 Netlify

**netlify.toml 配置**:
```toml

  publish = "dist"
  command = "npm run build"

  to = "/index.html"
```

## 🌍 方案三：GitHub Pages + Render

### 前端部署到 GitHub Pages

**步骤**:
1. 在 GitHub 仓库设置中启用 GitHub Pages
2. 选择 `main` 分支和 `/mobile-frontend/dist` 目录
3. 访问：`https://your-username.github.io/your-repo`

**GitHub Actions 自动部署**:
```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      
      - name: Install dependencies
        run: cd mobile-frontend && npm install
      
      - name: Build
        run: cd mobile-frontend && npm run build
      
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./mobile-frontend/dist
```

## 📋 部署前准备清单

### 1. 代码优化

**后端**:
```bash
# 1. 添加 .env 文件示例
cp .env.example .env

# 2. 添加 .gitignore
echo "node_modules/" >> .gitignore
echo ".env" >> .gitignore
echo "data/*.db" >> .gitignore
echo "uploads/*" >> .gitignore
echo "qrcodes/*" >> .gitignore

# 3. 添加 Procfile（Render 需要）
echo "web: node src/index.js" > Procfile

# 4. 添加 render.yaml
cat > render.yaml << EOF
services:
  - type: web
    name: distribution-backend
    env: node
    buildCommand: npm install
    startCommand: node src/index.js
    envVars:
      - key: NODE_ENV
        value: production
      - key: PORT
        value: 3010
EOF
```

**前端**:
```bash
# 1. 添加环境变量配置
cat > mobile-frontend/.env.production << EOF
VITE_API_BASE=https://your-app.onrender.com/api
EOF

# 2. 配置 vite.config.js
export default defineConfig({
  base: '/',  // GitHub Pages 需要改为 /repo-name/
  build: {
    outDir: 'dist',
    assetsDir: 'static'
  }
})
```

### 2. 数据库迁移

**SQLite 迁移**:
```bash
# 1. 导出当前数据
cd backend
sqlite3 data.db ".dump" > data/backup.sql

# 2. 生产环境初始化
# Render 会自动执行
node src/scripts/init-db.js
```

**PostgreSQL 迁移**（推荐）:
```bash
# 1. 安装 pg 驱动
npm install pg pg-hstore

# 2. 修改数据库配置
const sequelize = new Sequelize(
  process.env.DATABASE_URL,
  {
    dialect: 'postgres',
    dialectOptions: {
      ssl: {
        require: true,
        rejectUnauthorized: false
      }
    }
  }
);
```
