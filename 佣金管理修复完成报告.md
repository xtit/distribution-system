## 🐛 问题描述

用户反馈：管理后台佣金管理加载失败。

**错误信息**:
```
column Commission.commission_no does not exist
```

---

## 🔍 问题诊断

### 1. 检查数据库结构

线上 Supabase 数据库的 `commissions` 表缺少以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| ❌ commission_no | VARCHAR(50) | 佣金编号 |
| ❌ commission_rate | DECIMAL(5,2) | 佣金比例 |
| ❌ level | INTEGER | 分销层级 |
| ❌ description | VARCHAR(500) | 描述 |
| ❌ confirmed_at | TIMESTAMP | 确认可提现时间 |

### 2. 原因分析

本地 SQLite 数据库和线上 PostgreSQL 数据库结构不一致：
- 本地开发时模型定义了这些字段
- 但线上数据库没有同步这些字段
- Sequelize 查询时找不到字段，导致 500 错误

---

## ✅ 修复方案

### 执行数据库迁移脚本

创建并运行 `fix-commissions-table.js`：

```javascript
// 添加缺失的字段
ALTER TABLE commissions ADD COLUMN IF NOT EXISTS commission_no VARCHAR(50);
ALTER TABLE commissions ADD COLUMN IF NOT EXISTS commission_rate DECIMAL(5,2);
ALTER TABLE commissions ADD COLUMN IF NOT EXISTS level INTEGER DEFAULT 1;
ALTER TABLE commissions ADD COLUMN IF NOT EXISTS description VARCHAR(500);
ALTER TABLE commissions ADD COLUMN IF NOT EXISTS confirmed_at TIMESTAMP WITH TIME ZONE;
```

### 修复结果

✅ 所有字段添加成功  
✅ 唯一约束添加成功  
✅ 表结构完整
