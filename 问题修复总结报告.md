### 6️⃣ 数据统计 Bug 修复

**问题**: 首页销售额统计为 0（实际有订单）

**原因**: 销售额只统计 `completed` 和 `paid` 状态，遗漏了 `shipped` 状态

**修复**:
- ✅ 销售额统计添加 `shipped` 状态
- ✅ 商品统计添加 `paranoid: true` 排除已删除商品

**修改前**:
```javascript
where: { status: { [Op.in]: ['completed', 'paid'] } }
```

**修改后**:
```javascript
where: { status: { [Op.in]: ['paid', 'shipped', 'completed'] } }
```

**效果**: 销售额从 ¥0 变为 ¥718（正确）

**文件**: `backend/src/controllers/adminController.js`

---

### 7️⃣ 移动端确认收货功能修复

**问题**: 点击"确认收货"后订单状态不变

**原因**: 前端只调用了 `showDialog` 和 `showToast`，没有调用后端 API

**修复**:
- ✅ API 层添加 `confirmReceive()` 函数
- ✅ 视图层调用后端 API
- ✅ 优化提示信息（说明佣金冻结机制）

**修改前**:
```javascript
const confirmReceive = async (orderId) => {
  await showDialog({...})
  showToast('确认收货成功')  // ❌ 假成功
}
```

**修改后**:
```javascript
const confirmReceive = async (orderId) => {
  await showDialog({...})
  await confirmReceive(orderId)  // ✅ 调用 API
  showToast('确认收货成功')
}
```

**文件**:
- `mobile-frontend/src/api/product.js` (修改)
- `mobile-frontend/src/views/OrderList.vue` (修改)
