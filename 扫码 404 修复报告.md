## 🐛 问题描述

用户反馈：扫描二维码后，显示 Vercel 的 404 错误页面。

**错误信息**:
```
404: NOT_FOUND
Code: NOT_FOUND
ID: hkg1::ggxqw-1774338455952-073d28b05c69
```

---

## 🔍 问题诊断

### 原因分析

**Vercel SPA 路由配置问题**:

1. **前端框架**: Vue 3 + Vue Router（单页应用 SPA）
2. **部署平台**: Vercel
3. **问题**: Vercel 默认将每个请求当作静态文件处理
4. **现象**: 
   - 访问首页 `/` → ✅ 正常
   - 访问 `/register?ref=xxx` → ❌ 404（Vercel 找不到这个文件）

### 为什么会 404？

```
用户扫码 → 访问 /register?ref=ADMIN001
       ↓
Vercel 服务器查找 /register 文件
       ↓
❌ 文件不存在 → 返回 404
```

**正确流程应该是**:
```
用户扫码 → 访问 /register?ref=ADMIN001
       ↓
Vercel 返回 index.html
       ↓
Vue Router 接管路由 → 显示注册页面 ✅
```

---

## ✅ 修复方案

### 添加 Vercel 配置文件

**文件**: `mobile-frontend/vercel.json`

**内容**:
```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

### 配置说明

- **rewrites**: 重写规则
- **source**: 匹配所有路径 `/(.*)`
- **destination**: 都返回 `/index.html`
- **效果**: 让 Vue Router 处理所有前端路由

## 🎯 技术要点

### Vue Router + Vercel 部署

**问题**: Vercel 是静态托管，Vue Router 是客户端路由

**解决**: 配置 `vercel.json` rewrites 规则

**原理**:
```
所有请求 → Vercel → 返回 index.html → Vue Router → 显示对应页面
```

### 其他部署平台的配置

#### Netlify
创建 `netlify.toml`:
```toml
[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

#### Nginx
```nginx
location / {
  try_files $uri $uri/ /index.html;
}
```

#### Apache
创建 `.htaccess`:
```apache
<IfModule mod_rewrite.c>
  RewriteEngine On
  RewriteBase /
  RewriteRule ^index\.html$ - [L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule . /index.html [L]
</IfModule>
```
