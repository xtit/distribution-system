## 🐛 问题描述

用户反馈：商品编辑页面，图片上传后预览看不到图片。

## 🔍 问题分析

### 问题现象

1. 图片上传成功（后端已接收文件）
2. 后端返回 URL：`/uploads/image-xxx.png`
3. 前端显示了预览区域
4. **但图片不显示（空白）**

### 根本原因

**相对路径问题**：

后端返回的是相对路径：
```json
{
  "data": {
    "url": "/uploads/image-1774063090746-640141708.png"
  }
}
```

前端直接使用这个 URL：
```javascript
imgElement.src = url;  // ❌ /uploads/image-xxx.png
```

**问题**：浏览器会尝试从当前页面的域名加载图片，但可能因为以下原因失败：
1. 当前页面路径不是根路径（如 `/admin-frontend/product-edit.html`）
2. 相对路径解析错误
3. 没有错误处理，无法知道图片是否加载失败

## ✅ 修复方案

### 1️⃣ 转换为绝对路径

将后端返回的相对路径转换为完整的 URL：

```javascript
// 检查 URL 是否为相对路径
if (url && !url.startsWith('http://') && !url.startsWith('https://') && !url.startsWith('data:')) {
  // 拼接完整的 URL
  imageUrl = API_BASE.replace('/api', '') + url;
  // 结果：http://localhost:3010/uploads/image-xxx.png
}
```

### 2️⃣ 添加图片加载错误处理

```javascript
// 设置图片加载错误处理
imgElement.onerror = function() {
  console.error('图片加载失败:', imageUrl);
  showToast('图片加载失败，请检查图片链接', 'error');
};

// 设置图片加载成功处理
imgElement.onload = function() {
  console.log('图片加载成功:', imageUrl);
};
```

### 3️⃣ 添加调试日志

```javascript
console.log('显示图片预览，URL:', url);
console.log('转换为完整 URL:', imageUrl);
console.log('图片元素:', imgElement);
```

### 4️⃣ 空值检查

```javascript
// 确保元素存在
if (previewElement) previewElement.style.display = 'block';
if (uploadTextElement) uploadTextElement.style.display = 'none';
const uploadArea = document.querySelector('.image-upload-area');
if (uploadArea) uploadArea.classList.add('has-image');
```

```javascript
function showImagePreview(url) {
  // ✅ 转换为完整 URL
  let imageUrl = url;
  if (url && !url.startsWith('http://') && !url.startsWith('https://')) {
    imageUrl = API_BASE.replace('/api', '') + url;
  }
  
  // ✅ 添加错误处理
  imgElement.onerror = function() {
    console.error('图片加载失败:', imageUrl);
    showToast('图片加载失败', 'error');
  };
  
  // ✅ 添加成功回调
  imgElement.onload = function() {
    console.log('图片加载成功:', imageUrl);
  };
  
  imgElement.src = imageUrl;
}
```

## ⚠️ 注意事项

### 1. API_BASE 配置

确保 `API_BASE` 配置正确：
```javascript
const API_BASE = 'http://localhost:3010/api';
```

### 2. CORS 配置

如果前端和后端不在同一域名，需要配置 CORS：
```javascript
// 后端已配置
app.use(cors());
```

### 3. 静态文件服务

确保后端配置了静态文件服务：
```javascript
app.use('/uploads', express.static(path.join(__dirname, '../uploads')));
```

### 4. 图片路径验证

上传成功后，可以在浏览器直接访问图片 URL 验证：
```
http://localhost:3010/uploads/image-xxx.png
```

## 🔧 调试技巧

### 1. 检查图片是否可访问

在浏览器地址栏直接访问图片 URL：
```
http://localhost:3010/uploads/image-1774063090746-640141708.png
```

### 2. 检查控制台网络请求

1. 打开开发者工具
2. 切换到 Network 标签
3. 上传图片
4. 查看图片请求：
   - 状态码应该是 200
   - Content-Type 应该是 `image/png` 或 `image/jpeg`

### 3. 检查图片元素

在控制台执行：
```javascript
const img = document.getElementById('imagePreviewImg');
console.log('图片 src:', img.src);
console.log('图片完整 HTML:', img.outerHTML);
```
