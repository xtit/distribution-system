## 🐛 问题描述

用户反馈：管理后台商品编辑页面的图片上传功能有问题。

## 🔍 问题分析

### 可能的问题原因

1. **错误处理不完善**: 没有详细的错误日志，难以定位问题
2. **缺少调试信息**: 无法确认 API 请求是否发送成功
3. **Input 未清空**: 重复选择同一文件时可能不触发 onchange 事件
4. **Content-Type 问题**: FormData 上传时不应手动设置 Content-Type

### 原代码问题

```javascript
// 问题 1: 没有 try-catch 包裹整个函数
async function handleImageUpload(event) {
  const file = event.target.files[0];
  if (!file) return;  // 没有日志
  
  // 问题 2: 没有调试信息
  const res = await fetch(`${API_BASE}/admin/upload`, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${authToken}`
    },
    body: formData
  });
  
  // 问题 3: 没有清空 input
  // 重复选择同一文件时 onchange 不会触发
}
```

## ✅ 修复方案

### 1️⃣ 完善错误处理

将整个函数包裹在 try-catch 中：

```javascript
async function handleImageUpload(event) {
  try {
    // 所有代码
  } catch (error) {
    console.error('上传错误:', error);
    showToast('上传错误：' + error.message, 'error');
  } finally {
    // 清空 input
    event.target.value = '';
  }
}
```

### 2️⃣ 添加调试日志

```javascript
console.log('选择的文件:', file);
console.log('开始上传，文件名:', file.name, '大小:', (file.size / 1024).toFixed(2), 'KB');
console.log('API_BASE:', API_BASE);
console.log('Auth Token:', authToken ? '存在' : '不存在');
console.log('响应状态:', res.status);
console.log('响应数据:', data);
console.log('上传成功，URL:', imageUrl);
```

### 3️⃣ 清空 Input

```javascript
finally {
  // 清空 input，允许重复选择同一文件
  event.target.value = '';
}
```

### 4️⃣ 移除 Content-Type 设置

```javascript
// ❌ 错误做法
headers: {
  'Authorization': `Bearer ${authToken}`,
  'Content-Type': 'multipart/form-data'  // 不要手动设置！
}

// ✅ 正确做法
headers: {
  'Authorization': `Bearer ${authToken}`
  // 让浏览器自动设置 Content-Type（包含 boundary）
}
```

**原因**: FormData 上传时需要设置 `Content-Type: multipart/form-data; boundary=----WebKitFormBoundary...`，boundary 是随机生成的分隔符。如果手动设置，会缺少 boundary 导致后端无法解析。

### 5️⃣ 改进用户提示

```javascript
// 更详细的错误提示
showToast('请选择图片文件 (JPG/PNG)', 'error');
showToast('图片大小不能超过 5MB', 'error');
showToast('上传失败：' + data.message, 'error');
```

### 常见问题排查

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 没有日志输出 | 事件未触发 | 检查 input 的 onchange 绑定 |
| Token 不存在 | 登录失败 | 检查登录逻辑 |
| 404 错误 | API 地址错误 | 检查 API_BASE 配置 |
| 401 错误 | Token 无效 | 重新登录 |
| 400 错误 | 文件格式错误 | 检查文件类型和大小 |
| 500 错误 | 服务器错误 | 检查后端日志 |
