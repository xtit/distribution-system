## 🔄 数据流程

### 删除商品（软删除）
```
用户点击删除 
  → 确认弹窗 
  → DELETE /api/admin/products/:id 
  → 设置 deletedAt = 当前时间 
  → 商品从商品列表消失 
  → 可在回收站查看
```

### 恢复商品
```
用户点击恢复 
  → 确认弹窗 
  → POST /api/admin/products/:id/restore 
  → 设置 deletedAt = null, status = 1 
  → 商品重新出现在商品列表
```

### 彻底删除
```
用户点击彻底删除 
  → 二次确认（红色警告） 
  → DELETE /api/admin/products/:id/delete-forever 
  → 数据库物理删除记录 
  → 不可恢复
```

## ⚠️ 注意事项

1. **软删除机制**：
   - 删除商品不会立即从数据库删除
   - 商品会被标记为已删除（deletedAt 字段）
   - 可在回收站恢复

2. **彻底删除**：
   - 彻底删除操作不可恢复
   - 会有二次确认弹窗
   - 数据会永久从数据库移除

3. **商品列表**：
   - 商品列表只显示未删除的商品
   - 已删除商品自动过滤

4. **数据库兼容**：
   - 已自动执行迁移脚本添加 deleted_at 字段
   - 无需手动修改数据库
